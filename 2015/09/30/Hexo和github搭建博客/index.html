Hexo和github搭建博客

发表于 2015-09-30   |   分类于 开发

1.配置环境

安装Node 作用：用来生成静态页面
方法1：

  • 到Node.js官网下载相应平台的最新版本，一路安装即可

方法2：

  • 打开终端 将下面的命令 复制 输入终端中
  • 先下载Homebrew
  • 然后执行 brew install nodejs 安装nodejs 申请Github 作用：是用来做博客的远程仓库、域名、服务器之类的

2.正式安装Hexo

Node安装好后，先创建一个文件夹，如blog，用来存放hexo的配置文件，然后进入blog里安装Hexo.

  • 在终端里进入 blog 文件夹 我一般是先输入cd然后把文件夹拖入终端
  • 执行如下命令安装Hexo
  • npm install -g hero-cli 或 npm install -g hexo
  • 如果出现错误的话可以尝试一下npm install hexo –no-optional 如果还是有一点错误的话，错误不影响正常的博客写作和上传
  • 然后执行 init命令初始化
  • hexo init
  • 成功后打开文件夹，就可以看到一大堆文件了。这个文件夹就是你的博客的根目录

3.配置Github

建立新的repository, 在个人主页的右边，有一个New repository的绿色按钮，点一下
建立与你用户名对应的仓库，仓库名必须为

  • your_user_name.github.io,

比如我的就是

  • allenzlf.github.io

然后建立关联
打开 blog 文件夹下的 _config.yml 文件
到最下面，改成下面的样子

  • deploy:

    type: git

    repo: https://github.com/CoderMoe/codermoe.github.io.git

    branch: master

注意：冒号后面有空格， 否则会报错

然后执行命令：

  • npm install hexo-deployer-git –save 用来安装git上传的插件

再然后就执行： hexo d

会提示你输入github的用户名和密码
注意是github的用户名，不是你的仓库的名字
这样博客的基本配置就搭建好了

4.主要命令,写博客时用到

生成新的文章
hexo new 文章标题 如果文章标题含有空格使用 hexo new “文章标题”，生成的文章在blog/source/_posts文件夹里

清空已经生成的静态页面(会删除博客目录下的public文件夹) （有时候如果hexo d上传的时候出现下面的情况，那先hexo clean 再依次执行下面的步骤就会好了）

hexo clean
  • On branch master
  • nothing to commit, working directory clean
  • Everything up-to-date

生成静态页面（如果没有public文件夹的话，会生成这个文件夹）

hexo g (全称是 hexo generate)

在本地服务器查看

hexo s (全称是 hexo server) 然后在浏览器输入 http://localhost:4000

上传到 github

hexo d (全称是 hexo deploy)

5.主题配置

这个是按列表展示主题

  • https://github.com/hexojs/hexo/wiki/Themes

在git里面下载的话会比较好,自我感觉比较方便
然后移到博客根目录里的themes文件夹里。将主题文件名复制一下，打开_config.yml，翻到下面找到 theme:将原来的主题名称替换掉。需要注意冒号后面有一个空格 然后在终端输入hexo g就生成静态页面了。
